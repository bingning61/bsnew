# 论文主上下文文档

## 一、项目名称与一句话定义

为便于论文写作，建议将当前项目表述为：

**基于 YOLO 视觉感知与中心偏差控制的 ROS1 焊缝跟踪机器人系统**

一句话定义为：

该系统利用相机图像中的焊缝目标检测结果，提取检测框中心作为目标位置表征量，并通过保留的中心偏差控制器生成 `cmd_vel`，从而在 ROS1 平台上实现焊缝目标的视觉闭环跟踪。

## 二、研究背景与问题来源

从当前仓库真实代码结构看，项目的技术起点包含两类基础能力：一类是 `yolo/` 目录中的目标检测能力，另一类是 `robot_vision` 与 `base_control` 中已经可运行的 ROS1 视觉控制与底盘执行链路。原始控制链路更接近“依据图像中线中心位置进行偏差修正”的实现，而当前项目要面向的是焊缝目标跟踪任务。

因此，项目面对的实际问题并不是“如何重新设计一套复杂的机器人控制系统”，而是“如何将焊缝目标的视觉识别结果，转化为现有移动机器人控制链路可直接使用的偏差量，并在 ROS1 环境下实现稳定闭环”。这一问题具有明显的本科毕业设计特征：目标清晰、逻辑链条短、实现可落地、工程风险可控，同时又能在论文层面提炼为完整的方法体系。

## 三、本项目要解决的核心问题

结合当前源码，可以把核心问题概括为四个连续子问题：

1. 焊缝目标如何在相机图像中被识别出来。
2. 检测结果如何转化为控制器可理解的位置量。
3. 位置量如何与期望位置建立偏差关系。
4. 偏差如何进一步转化为机器人前进与转向速度。

因此，本项目本质上是在解决“视觉目标位置到运动控制量的映射问题”，并且是在保留原有控制链路的前提下完成这种映射。

## 四、当前系统总体功能

根据当前主入口 `catkin_ws/src/robot_vision/launch/seam_tracking.launch`，系统已经具备以下总体功能：

- 从真实相机或假相机获得图像输入；
- 使用 YOLO 节点进行目标检测；
- 从检测框中提取目标中心横坐标；
- 将目标中心与图像中心建立偏差；
- 依据偏差生成 `cmd_vel`；
- 在需要时接入 `base_control` 底盘接口；
- 在无有效检测或中心点超时的情况下执行停机；
- 输出调试图像用于识别结果可视化。

由此可以看出，当前系统已经形成了一条完整的“感知 -> 表征 -> 偏差 -> 控制 -> 执行”工程链路。

## 五、当前项目采用的总体技术路线

当前项目的总体技术路线可以概括为：

**基于目标检测中心的兼容式偏差控制路线。**

其核心思想不是重新构建一套全新的焊缝控制器，而是把焊缝检测结果压缩成原控制器可以直接使用的“中心位置代理量”，再通过已有的偏差控制逻辑输出速度指令。换言之，项目真正提出的是一种“以检测框中心替代传统线中心”的焊缝跟踪实现路线。

这条路线在论文中可以被展开为以下方法链：

1. 基于 YOLO 检测的焊缝目标感知；
2. 基于检测框中心的目标位置表征；
3. 基于图像中心参照的偏差构建；
4. 基于归一化偏差的焊缝跟踪控制；
5. 基于转向强度约束的速度协同调节；
6. 基于 ROS1 话题解耦的感知到控制闭环实现；
7. 基于有效性标志与超时监测的异常安全处理。

## 六、当前项目的主要方法体系

### 1. 焊缝目标感知方法

系统使用 `robot_vision/scripts/yolo_seam_detector.py` 完成图像目标检测。节点订阅 `/image_raw`，调用本地 `yolo/ultralytics` 中的 `YOLO` 类执行推理，在多候选框情况下按置信度选择最佳框，并支持 `target_class_id` 进行类别约束。

### 2. 目标位置表征方法

系统不把完整检测框直接交给控制器，而是提取检测框中心横坐标，并将其编码为 `geometry_msgs/Point`。其中：

- `Point.x` 表示目标中心横坐标；
- `Point.y` 表示图像宽度；
- `Point.z` 表示检测是否有效。

这种位置表征方式本质上是一种轻量几何代理方法。

### 3. 偏差构建方法

控制器把图像中心 `W/2` 作为期望位置，把目标中心横坐标 `c_x` 作为实际位置，从而形成横向偏差。论文中可将其写为：

```text
e_px = c_x - W/2
```

而源码中的控制逻辑实际采用符号相反的控制表达：

```text
e_ctrl = W/2 - c_x
```

### 4. 跟踪控制方法

系统保留了原有 `line_detector.py` 的 `twist_calculate()` 控制函数。该函数依据偏差计算 `angular.z`，再按转向强度调节 `linear.x`。当目标处于中心附近时，机器人直行；当偏差增大时，角速度增加、线速度降低。

### 5. 速度协同调节方法

当前实现采用分段式协同调节：目标居中时快速直行，小偏差时速度与角速度联动，大偏差时降低到保守速度。这使系统兼顾了跟踪响应与运动稳定性。

### 6. ROS1 闭环实现方法

系统通过 `seam_tracking.launch` 将相机、YOLO 节点、控制节点和底盘接口组织在同一条 ROS1 话题链路上，实现视觉到运动的反馈闭环。

### 7. 异常安全处理方法

当前实现中，当检测无效或中心点超时时，控制器会发布零速度；底盘协议文档还定义了上位机断联后的主动停机机制。这构成了当前系统的基础安全策略。

## 七、当前项目的主要原理与逻辑

当前系统最关键的原理可以用一句话概括：

**把焊缝目标在图像中的横向位置偏差，转换成机器人前进速度与转向角速度。**

展开后，整个逻辑如下：

1. 相机获取当前作业画面；
2. YOLO 检测器在图像中找到焊缝目标区域；
3. 节点提取检测框中心横坐标 `c_x`；
4. 系统计算图像中心 `W/2`；
5. 两者形成横向偏差；
6. 控制器依据偏差输出 `cmd_vel`；
7. 底盘执行后，焊缝在下一帧中的位置发生变化；
8. 进入下一轮视觉与控制。

这条逻辑链条完整回答了“视觉信息如何变成运动控制量”这一论文核心问题。

## 十四、当前系统的局限性与后续改进方向

### 1. 当前局限性

- 目标位置表征仅使用检测框中心，缺少更丰富的几何信息；
- 控制方法为经验式比例控制，未引入高阶控制；

- 增加焊缝方向、长度或多点位置特征，提高位置表征能力；
- 引入时序滤波或多帧稳定机制；
- 对控制参数开展系统化实验标定；
- 建立焊缝场景下的 Gazebo 仿真闭环；

## 十五、回到原运行环境后的构建与启动方式

### 1. 构建方式

```bash
cd ~/bsnew/catkin_ws
source /opt/ros/melodic/setup.bash
catkin_make
source devel/setup.bash
```

如果使用其他 ROS1 发行版，则把 `melodic` 替换为对应发行版名称。

### 2. 推荐的调试启动方式

由于当前仓库没有自带 `.pt` 权重文件，也没有实机环境变量说明，因此最稳妥的调试方式是先使用假相机：

```bash
roslaunch robot_vision seam_tracking.launch \
  run_base_control:=false \
  use_fake_camera:=true \
  fake_image_path:=$(rospack find robot_vision)/data/bingda.png \
  model_path:=<YOLO_WEIGHT_PATH>
```

### 3. 实机启动方式的源码级要求

根据当前源码，实机启动前还应人工确认：

```bash
export BASE_TYPE=<待确认>
export CAMERA_TYPE=<待确认，可能为 csi72 或 astrapro>
```

之后再执行：

```bash
roslaunch robot_vision seam_tracking.launch \
  run_base_control:=true \
  model_path:=<YOLO_WEIGHT_PATH>
```

其中 `<YOLO_WEIGHT_PATH>` 必须由人工替换成实际存在的权重文件路径。README 与代码注释中出现过 `../yolo/runs/train/exp17/weights/best.pt`，但该文件并不在当前仓库中。

## 十六、必须明确回答的关键问题

### 1. 这个项目最终是什么系统？

它是一个基于 YOLO 感知前端与中心偏差控制后端的 ROS1 焊缝跟踪机器人系统。

### 2. 这个系统的研究目标是什么？

研究目标是实现从焊缝视觉检测到机器人运动控制的完整闭环。

### 3. 这个系统采用了哪些具体方法？

采用了目标检测、中心位置表征、图像中心偏差构造、分段式速度控制、ROS1 话题闭环和异常停机等方法。

### 4. 每个方法的原理是什么？

核心原理是把检测框中心当作焊缝位置代理量，再以其相对图像中心的偏差驱动控制器。

### 5. 系统完整逻辑链条是什么？

图像输入 -> YOLO 检测 -> 检测框中心提取 -> 偏差构建 -> `cmd_vel` 生成 -> 底盘执行 -> 新图像反馈。

### 6. YOLO 在这里承担什么功能？

承担焊缝目标感知与目标中心提取前的检测任务。

### 7. 控制器在这里承担什么功能？

承担偏差到速度的映射任务，输出机器人前进与转向控制量。

### 8. 视觉信息是如何转换成运动控制量的？

通过“检测框中心 -> 中心偏差 -> `Twist` 输出”这条变换链实现。

### 9. 系统闭环是如何形成的？

底盘运动改变视野中的目标位置，新的图像再次进入感知节点，从而形成反馈闭环。
