# 论文缺失信息清单

## 使用说明
本清单用于回答一个现实问题：**当前仓库已经足够支撑“系统设计与实现”写作，但还不够支撑完整毕业论文定稿。**  
下面列出的每一项都说明了：

1. 为什么论文可能需要它。
2. 当前仓库里有没有。
3. 如果没有，你后续应该怎么补。

## 建议你优先补的五项

1. 真实 YOLO 权重文件与类别说明。
2. Ubuntu 18.04 + ROS1 的成功构建与运行截图。
3. 实际底盘型号、相机型号、`BASE_TYPE`、`CAMERA_TYPE`。
4. `/image_raw`、`/seam_center`、`cmd_vel`、`odom` 的话题截图。
5. 至少一组实机或半实机实验结果截图与简要统计表。

## 如果你时间很紧，最低限度也要补什么
如果你希望尽快把论文写出来，但时间有限，至少建议补下面这些最低限度材料：

1. 一张系统总体结构图。
2. 一张 `roslaunch robot_vision seam_tracking.launch` 运行截图。
3. 一张 YOLO 检测结果图。
4. 一张实机或模拟运行场景图。
5. 一张简单实验结果表，例如“是否检测到目标 / 是否输出 `cmd_vel` / 是否停车成功”。

有了这些最低限度材料，论文的“系统实现”和“实验说明”部分会更完整，也更不容易被答辩老师质疑。
