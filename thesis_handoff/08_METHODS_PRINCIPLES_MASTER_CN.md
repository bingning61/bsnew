# 方法、原理、思路与逻辑主文档

## 1. 本项目最终系统的一句话技术定义

当前项目最终可以定义为：**一种基于 YOLO 焊缝目标检测、检测框中心位置表征与图像中心偏差控制的 ROS1 机器人闭环跟踪系统**。

这一定义包含了三个关键词。第一是“YOLO 焊缝目标检测”，说明视觉前端采用目标检测获得焊缝目标区域。第二是“检测框中心位置表征”，说明系统并不直接把检测框原样交给控制器，而是抽取中心横坐标作为控制几何量。第三是“图像中心偏差控制”，说明控制核心围绕目标中心与图像中心的偏差展开，而不是重新构造一套完全不同的运动控制框架。

## 2. 本项目面向的核心问题

本项目面向的核心问题不是“如何把两个系统接起来”，而是：**如何把视觉检测结果组织成控制器能够直接使用的位置偏差量，并在 ROS1 环境下形成从感知到执行的稳定闭环**。

如果进一步拆解，这个核心问题包含四个子问题：

1. 如何从图像中识别焊缝目标。
2. 如何从识别结果中提取可用于控制的几何位置量。
3. 如何把目标位置与期望位置建立偏差关系。
4. 如何根据偏差生成运动控制量并实现安全闭环。

因此，论文的技术主线应被表述为“感知目标、表征位置、构造偏差、输出控制、形成闭环、验证可行性”。

## 3. 本项目采用的总体技术路线

结合当前代码，本项目的总体技术路线可写为：

**图像输入 -> YOLO 目标检测 -> 最优检测框筛选 -> 检测框中心提取 -> 图像中心参考构造 -> 偏差构建 -> 速度协同调节 -> `cmd_vel` 输出 -> 底盘执行 -> 下一帧图像反馈**

在 ROS1 实现上，这一路线由 `seam_tracking.launch` 组织完成。相机或假相机节点提供 `/image_raw`；`yolo_seam_detector.py` 负责从图像中得到控制所需的目标位置量；`line_detector.py` 负责从目标位置量构造偏差并生成 `cmd_vel`；`base_control.py` 负责把 `cmd_vel` 下发到底盘执行。当前系统还通过无效检测与超时监测机制实现安全停止。

这一路线适合本科毕业设计的关键原因在于：视觉与控制通过最小几何接口耦合，逻辑清楚，改动有限，便于调试与论文化表达。

## 4. 本项目的核心方法体系

根据当前仓库，最适合抽象为以下八种方法：

1. 基于 YOLO 的焊缝目标感知与最优检测框筛选方法
2. 基于检测框中心的目标位置表征方法
3. 基于图像中心参考的偏差构建方法
4. 基于归一化偏差的角速度生成方法
5. 基于偏差大小的线速度协同调节方法
6. 基于 ROS1 话题适配的感知到控制闭环实现方法
7. 基于有效标志与超时监测的异常处理与安全停止方法
8. 基于 URDF、Gazebo 与 Stage 的模型与仿真支撑方法

下面分别展开说明。

## 5. 方法一：基于 YOLO 的焊缝目标感知与最优检测框筛选方法

### 5.1 方法名称

基于 YOLO 的焊缝目标感知与最优检测框筛选方法。

### 5.2 要解决的问题

该方法要解决的问题是：在每一帧输入图像中，如何快速找到焊缝目标区域，并在多个候选检测框中确定一个最适合控制器使用的目标框。

### 5.3 输入

输入是 `/image_raw` 对应的彩色图像，以及模型路径、置信度阈值、输入尺寸和目标类别筛选参数等运行参数。

### 5.4 输出

输出是一个最优检测框，以及与之关联的置信度和类别编号；如果没有有效检测框，则输出“无有效目标”的状态。

### 5.5 基本原理

系统通过 `model.predict()` 对输入图像进行目标检测，得到一组候选框。随后，对每个候选框读取置信度和类别编号，若设置了 `target_class_id`，则先进行类别过滤；在通过过滤的候选框中，选择置信度最高的检测框作为当前控制目标。

从方法原理上讲，这是一种“检测后单目标筛选”的策略。它的目标不是保留全部视觉信息，而是从控制需求出发，获得当前最有代表性的目标区域。

### 5.6 关键逻辑关系或公式关系

若候选检测框集合记为 `B={b_i}`，每个检测框的置信度记为 `c_i`，则最优检测框满足：

`b* = arg max(c_i)`

当 `target_class_id>=0` 时，仅在满足类别筛选条件的框中求取该最大值。

### 5.7 在当前项目中由哪些代码实现

该方法由以下文件实现：

- `catkin_ws/src/robot_vision/scripts/yolo_seam_detector.py`
- `catkin_ws/src/robot_vision/launch/seam_tracking.launch`

其中，`find_best_box()` 负责候选框筛选，`image_callback()` 负责整帧图像的推理与检测结果处理。

### 5.8 为什么适合当前本科毕业设计

该方法实现直接、代码结构清楚、接口简单，不需要引入复杂目标跟踪或多目标管理机制，能够把论文重点放在“目标检测如何服务闭环控制”这一核心问题上。

### 5.9 局限性

该方法默认选择置信度最高的单个目标框，因此在复杂场景、多目标干扰或目标短时遮挡情况下，可能出现目标切换或误选问题。当前仓库没有引入时序跟踪和多帧稳定机制。

## 6. 方法二：基于检测框中心的目标位置表征方法

### 6.1 方法名称

基于检测框中心的目标位置表征方法。

### 6.2 要解决的问题

该方法要解决的问题是：如何把视觉检测框转换为控制器真正需要的位置量，而不是把整套检测框几何信息直接传递给后端。

### 6.3 输入

输入是最优检测框坐标 `(x1, y1, x2, y2)` 以及当前图像宽度 `W`。

### 6.4 输出

输出是目标中心横坐标 `x_obj`、图像宽度 `W` 以及检测有效标志 `v`。

### 6.5 基本原理

系统以检测框中心横坐标作为目标位置的代表性几何量。这样做的理由在于：当前控制器只需要知道目标在图像中偏左还是偏右，以及偏差有多大，并不需要完整检测框参与控制计算。

### 6.6 关键逻辑关系或公式关系

目标中心横坐标定义为：

`x_obj = (x1 + x2) / 2`

在 ROS 接口中，系统把消息编码为：

- `Point.x = x_obj`
- `Point.y = W`
- `Point.z = v`

### 6.7 在当前项目中由哪些代码实现

该方法由 `yolo_seam_detector.py` 中的 `publish_center()` 与 `image_callback()` 实现。

### 6.8 为什么适合当前本科毕业设计

该方法最大优点是接口极简。它把视觉结果压缩为最小几何量，既降低了模块耦合，也让论文中的“位置表征方法”非常清晰。

### 6.9 局限性

检测框中心只能描述目标的大致横向位置，无法表达焊缝方向、角度、曲率或细粒度轮廓信息，因此适用于“中心对准”型控制，但不适用于高精度几何跟踪。

## 7. 方法三：基于图像中心参考的偏差构建方法

### 7.1 方法名称

基于图像中心参考的偏差构建方法。

### 7.2 要解决的问题

该方法要解决的问题是：如何把目标位置量转化为控制器可直接使用的偏差量。

### 7.3 输入

输入是目标中心横坐标 `x_obj` 和图像宽度 `W`。

### 7.4 输出

输出是目标中心相对于图像中心的偏差 `e`。

### 7.5 基本原理

系统把图像中心视为机器人理想对准焊缝目标时的参考位置。若目标中心偏离图像中心，则说明机器人当前需要做姿态修正。因此，图像中心相当于控制系统的期望值。

### 7.6 关键逻辑关系或公式关系

参考中心：

`x_ref = W / 2`

几何偏差可写为：

`Δx = x_obj - x_ref`

当前控制代码等价采用：

`e = x_ref - x_obj`

以及归一化形式：

`e_n = (x_ref - x_obj) / x_ref`

### 7.7 在当前项目中由哪些代码实现

该方法由 `line_detector.py` 中 `external_center_callback()` 调用 `twist_calculate(image_width / 2.0, center_x)` 的方式实现。

### 7.8 为什么适合当前本科毕业设计

该方法不依赖复杂标定和三维坐标恢复，仅利用图像宽度即可构建参考中心，逻辑简单、说明清楚，适合本科论文表达。

### 7.9 局限性

偏差构建基于二维图像坐标，未考虑目标距离、透视变化和相机姿态变化的影响，因此控制量更多体现为图像空间对准，而不是严格的空间几何对准。

## 8. 方法四：基于归一化偏差的角速度生成方法

### 8.1 方法名称

基于归一化偏差的角速度生成方法。

### 8.2 要解决的问题

该方法要解决的问题是：当目标偏离图像中心时，如何根据偏差大小与方向生成机器人转向角速度。

### 8.3 输入

输入是参考中心 `x_ref` 和目标中心 `x_obj`。

### 8.4 输出

输出是角速度 `ω = angular.z`。

### 8.5 基本原理

系统把目标偏差直接映射到转向角速度，偏差越大，转向修正越明显；偏差越小，转向修正越弱。这本质上是一种比例式偏差控制思想。

### 8.6 关键逻辑关系或公式关系

当前代码中的角速度为：

`ω = (x_ref - x_obj) / (2 x_ref)`

这说明角速度与归一化偏差成正比。

### 8.7 在当前项目中由哪些代码实现

该方法由 `line_detector.py` 中 `twist_calculate()` 实现。

### 8.8 为什么适合当前本科毕业设计

该方法结构简单、参数少、可解释性强，便于在论文中分析“偏差与控制量之间的因果关系”。

### 8.9 局限性

当前方法没有积分、微分、滤波或预测机制，因此对噪声抑制和动态性能的提升有限。

## 9. 方法五：基于偏差大小的线速度协同调节方法

### 9.1 方法名称

基于偏差大小的线速度协同调节方法。

### 9.2 要解决的问题

该方法要解决的问题是：如何在转向修正的同时，协调机器人前进速度，使系统兼顾跟踪稳定性和运行效率。

### 9.3 输入

输入是角速度 `ω` 或与之等价的归一化偏差量。

### 9.4 输出

输出是线速度 `v = linear.x`。

### 9.5 基本原理

系统采用分段式速度调节。当目标已基本对准时，保持较高前进速度；当偏差较大时，降低前进速度，为转向修正让出裕度；当偏差中等时，线速度按角速度做耦合调节。

### 9.6 关键逻辑关系或公式关系

若目标基本对准：

`0.95 < x_obj / x_ref < 1.05`

则：

`v = 0.2`

若目标偏离中心，则先计算 `ω`，再按如下规则调整：

- 当 `|ω| < 0.2` 时，`v = 0.2 - ω / 2`
- 当 `|ω| >= 0.2` 时，`v = 0.1`

### 9.7 在当前项目中由哪些代码实现

该方法由 `line_detector.py` 的 `twist_calculate()` 完成。

### 9.8 为什么适合当前本科毕业设计

该方法不需要复杂动力学模型，易于调试，且能直观展示“偏差越大速度越保守”的控制思路。

### 9.9 局限性

该方法的速度调节带有经验性，当前表达式还存在与角速度符号耦合的特征，是否最优仍需实机标定和测试。

## 10. 方法六：基于 ROS1 话题适配的感知到控制闭环实现方法

### 10.1 方法名称

基于 ROS1 话题适配的感知到控制闭环实现方法。

### 10.2 要解决的问题

该方法要解决的问题是：如何在不重写底盘执行链路的前提下，把视觉结果接入现有控制系统，形成完整闭环。

### 10.3 输入

输入是图像话题 `/image_raw` 和中心消息话题 `/seam_center`。

### 10.4 输出

输出是标准运动指令 `cmd_vel` 以及后续底盘执行动作。

### 10.5 基本原理

系统把视觉输出与控制输入之间的耦合，设计为标准 ROS1 话题传输。视觉节点只负责发布最小几何量，控制节点只负责消费该几何量并生成 `cmd_vel`，底盘节点再消费 `cmd_vel`。由此，感知、控制和执行三者职责分离。

### 10.6 关键逻辑关系或公式关系

可写成如下链路：

`/image_raw -> /seam_center -> cmd_vel -> serial chassis command`

### 10.7 在当前项目中由哪些代码实现

该方法由以下文件协同实现：

- `robot_vision/launch/seam_tracking.launch`
- `robot_vision/scripts/yolo_seam_detector.py`
- `robot_vision/scripts/line_detector.py`
- `base_control/launch/base_control.launch`
- `base_control/script/base_control.py`

### 10.8 为什么适合当前本科毕业设计

这种实现方式最大优点是工程清晰、可复用、低风险。论文可以把重点放在接口设计与闭环逻辑，而不是纠缠于大规模重构。

### 10.9 局限性

当前闭环主要依赖视觉中心偏差，没有引入 odom 到控制器的多源反馈融合，因此控制维度相对单一。

## 11. 方法七：基于有效标志与超时监测的异常处理与安全停止方法

### 11.1 方法名称

基于有效标志与超时监测的异常处理与安全停止方法。

### 11.2 要解决的问题

该方法要解决的问题是：当目标丢失、检测失败或话题中断时，系统如何避免继续盲目前进。

### 11.3 输入

输入是 `Point.z` 有效标志和中心消息接收时间。

### 11.4 输出

输出是零速度 `Twist()`，以及底盘停止。

### 11.5 基本原理

如果视觉节点未能检测到目标，则直接发布无效标志；如果控制节点在规定时间内未收到新的有效消息，则通过看门狗触发停止；即使上位机侧停止失效，下位机底盘协议仍有超时断连停机保护。

### 11.6 关键逻辑关系或公式关系

- 当 `v = 0` 时，发布 `Twist()` 零速度
- 当 `t_now - t_last > timeout` 时，发布 `Twist()` 零速度

### 11.7 在当前项目中由哪些代码实现

- `yolo_seam_detector.py` 负责发布有效/无效状态
- `line_detector.py` 的 `publish_stop()` 和 `external_center_watchdog()` 负责上层停止
- `base_control/README.md` 描述了下位机 1000ms 超时停机机制

### 11.8 为什么适合当前本科毕业设计

安全停止逻辑简单、必要且易于展示，能体现系统完整性和工程可靠性。

### 11.9 局限性

当前异常处理以保守停止为主，没有更复杂的目标重捕获或故障恢复策略。

## 12. 方法八：基于 URDF、Gazebo 与 Stage 的模型与仿真支撑方法

### 12.1 方法名称

基于 URDF、Gazebo 与 Stage 的模型与仿真支撑方法。

### 12.2 要解决的问题

该方法要解决的问题是：如何为论文第四章提供机器人模型、通用移动平台仿真和系统实现支撑，而不夸大其在主焊缝跟踪链路中的作用。

### 12.3 输入

输入是机器人结构描述、Gazebo/Stage 场景、地图和传感器插件配置。

### 12.4 输出

输出是机器人模型可视化、Gazebo/Stage 运行环境以及通用话题级仿真支撑。

### 12.5 基本原理

`nanoomni_description` 提供 URDF/Xacro 和 Gazebo 传感器插件，使机器人在仿真中具备 `cmd_vel`、`odom`、`image_raw`、`scan` 等接口；`robot_simulation` 提供 Stage 单车/双车与地图环境。二者共同为系统验证和后续扩展提供基础环境。

### 12.6 关键逻辑关系或公式关系

这里的关键关系不是公式，而是接口兼容：

- Gazebo 相机插件发布 `image_raw`
- Gazebo 平面运动插件使用 `cmd_vel` 与 `odom`
- Stage 环境发布 `scan` 与 `odom`

### 12.7 在当前项目中由哪些代码实现

- `catkin_ws/src/nanoomni_description/launch/*.launch`
- `catkin_ws/src/nanoomni_description/urdf/*.xacro`
- `catkin_ws/src/robot_simulation/launch/*.launch`
- `catkin_ws/src/robot_navigation/launch/*.launch`

### 12.8 为什么适合当前本科毕业设计

这些资源说明项目不仅有主控制链路，也有机器人模型和通用仿真基础，有助于第四章形成更完整的系统实现与分析结构。

### 12.9 局限性

当前仓库没有焊缝专用仿真场景和与 `seam_tracking.launch` 直接联调的专用仿真入口，因此不能把现有仿真资源直接等同于“焊缝跟踪仿真已经完成”。

## 13. 关键问答

### 13.1 这个项目最终是什么系统

它是一个基于 YOLO 目标检测、检测框中心位置表征和偏差驱动控制的 ROS1 焊缝跟踪机器人系统。

### 13.2 这个系统的研究目标是什么

研究目标是把视觉检测结果转化为可用于运动控制的几何偏差量，并利用现有 `cmd_vel` 链路实现焊缝目标闭环跟踪与安全停止。

### 13.3 YOLO 在这里承担什么功能

YOLO 承担视觉前端功能，负责从图像中检测焊缝目标，并输出最优检测框中心位置。

### 13.4 控制器在这里承担什么功能

控制器承担偏差构建和速度生成功能，负责把目标中心与图像中心之间的偏差转换为 `cmd_vel`。

### 13.5 视觉信息是如何转换成运动控制量的

系统先计算检测框中心，再以图像中心为参考构造偏差，最后通过 `twist_calculate()` 生成角速度和线速度。

### 13.6 系统闭环是如何形成的

机器人运动改变下一帧图像中的目标位置，新的图像 আবার进入 YOLO 检测与偏差控制环节，由此形成以视觉偏差为反馈量的闭环。

### 13.7 为什么这种方法路线适合本科毕业设计

因为它逻辑清楚、工程风险低、实现复杂度适中，同时可以抽象出完整的方法链条，便于论文化表达。

## 14. 本文档的结论

当前项目最适合被写成一套“基于视觉目标检测与中心偏差控制的焊缝跟踪方法体系”。在这套体系中，YOLO 不是孤立的识别模块，控制器也不是抽象黑箱，而是围绕“目标中心相对于图像中心的偏差”这一中间量建立起明确的因果链条。正因为如此，现有工程实现能够被提升为一篇结构完整、方法清晰、逻辑自洽的本科毕业论文。
